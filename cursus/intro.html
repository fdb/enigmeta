HTMLWTF Korte historiek van het internet

Het internet is fantastisch. Het is de grootste bron aan informatie over van alles en nog wat. Door computers van heel de wereld aan elkaar te schakelen tot een netwerk van servers kan je een fantastische wereld aan informatie binnenkrijgen.

Vandaag gaan we achter de schermen kijken van het web. Elke webpagina is opgemaakt in HTML en vormgegeven met CSS. Het gedrag wordt bepaald door JavaScript.

Hier zie je wat HTML code:

<div>
  <h1>Hoofding</h1>
  <p>Hier komt wat tekst...</p>
</div>

HTML bestaat uit gewone tekst inclusief tags. Dit is een tag: <html>. Je zal er in de cursus nog heel wat zien. Dit lijkt misschien beangstigend in het begin, maar we nemen rustig onze tijd om dit goed door te nemen.

Bottom-up Over de aanpak van de cursus

We hebben er speciaal voor gekozen om je HTML en CSS te leren zodat je echt weet hoe pagina's gemaakt worden, en wat de code erachter is. Dit is belangrijk om verschillende redenen.

Zelfredzaamheid

Je zou, in plaats van met HTML te werken, ook met een grafische editor werken. Die schrijft dan de HTML voor jou. Fantastisch!

Tot er iets mis gaat. De editor vormt een dun laagje boven de HTML en CSS: hij abstraheert als het ware de HTML en CSS tot een grafische omgeving.

Echter, door te doen alsof HTML en CSS niet bestaan loop je tegen problemen aan die je zelf niet kan oplossen. Een element staat niet op de juiste plaats? Als je nooit de HTML en CSS ziet weet je ook niet waar je moet beginnen zoeken.

Daarom is het belangrijk om te begrijpen hoe webpagina's gemaakt worden, alvorens je visuele tools gebruikt om pagina's aan te maken.

Samenwerking

In de "echte" wereld zal je grote websites waarschijnlijk in team maken. Als je samenwerkt met een programmeur die de HTML en CSS voor jou doet is het nog steeds belangrijk om te begrijpen wat er mogelijk is. Dit laat de conversatie tussen jou en andere teamleden vlotter verlopen.

Alternatieven Waarom leren we niet gewoon Flash?

Veel designer-sites zijn in Flash gemaakt. Flash is ideaal voor animatie, geluid, video, dus waarom doen we niet alles in Flash?

Toegankelijkheid

De hoofdreden is toegankelijkheid. Mensen die niet in staat zijn van visuele webpagina's te bekijken (zoals blinden) kunnen je Flash-website niet bezoeken. Dit is misschien niet belangrijk als je een portfolio met grafisch werk maakt, maar wordt wel belangrijk als je websites ontwikkeld in de "echte" wereld, waar je er niet kan van uitgaan dat mensen kunnen horen, zien, of ruiken.

Zoekmachines

Daarnaast zijn er de zoekmachines. Een site in Flash bevat bijna geen zinnige informatie voor zoekrobots, waardoor je website het erg povertjes zal doen in Google.

Een zoekmachine is eigenlijk ook "blind". Het kan niet kijken naar de pixels, maar enkel tekst lezen.

Mobiel

Tenslotte zijn er bezoekers die met een mobiele webbrowser de pagina bekijken, zoals de iPhone of iPad.

Deze apparaten ondersteunen geen Flash, en het is duidelijk uit Apple's standpunt dat dit in hun ogen een verouderde technologie is die ze niet willen ondersteunen.

Structuur van een webpagina

Een webpagina bestaat uit drie verschillende onderdelen:

Structuur: HTML

De structuur bepaalt de verschillende onderdelen van de inhoud zijn, en hoe ze in relatie staan. Dit is de taak van HTML. Het bevat de inhoud van je pagina en wat extra informatie om aan te duiden welke delen de hoofdingen zijn, welke paragrafen, welke tekst er moet benadrukt worden, enz.

Presentatie: CSS

De presentatie bepaalt hoe de inhoud visueel getoond moet worden. Dit is de taak van CSS. CSS selecteert onderdelen op de pagina en beschrijft de kleur, lettertype en spatiëring waarmee de elementen moeten worden getoond.

Gedrag: JavaScript

Het gedrag bepaalt hoe de inhoud reageert en verandert gebaseerd op interactie met de gebruiker. Dit is de taak van JavaScript. JavaScript is een volledige programmeertaal waarmee je het gedrag van een pagina volledig kan sturen. Het is krachtig en complex.

Waarom inhoud en opmaak scheiden?

In vroegere tijden, dat wil zeggen, 1998, werden structuur & presentatie gemengd. Dit leidde tot een grote soep aan code. Tegenwoordig hoeft dat allemaal niet. Daar zijn goede redenen voor:

Toegankelijker

Propere, semantische HTML heeft voordelen voor gebruikers die niet-visuele webbrowsers gebruiken, zoals blinden. Daarnaast zijn er ook zoekrobotten die je website opzoeken, en die kunnen ook niet zien. Google leest je site en gebruikt vooral de structuur-informatie uit HTML om te bepalen wat belangrijk is.

Onderhoudbaarder

Door HTML/CSS te scheiden kan je een nieuwe stylesheet implementeren en je hebt een mobiele versie van je pagina. We kunnen ook delen tussen verschillende projecten herbruiken. Een designer kan aanpassingen aan de CSS doen zonder de HTML te hoeven aan te passen of de JavaScript code te veranderen.

Sneller

De meeste websites bestaan uit meerdere pagina's. Door de opmaak te scheiden hoef je die maar één keer te downloaden voor alle pagina's. De browser bewaart daarna de opmaak lokaal, in een cache.

CSS Zen Garden

CSS Zen Garden toont dat één enkele HTML-pagina op duizenden verschillende manieren kan getoond worden. De HTML-broncode is altijd hetzelfde. Enkel de opmaak, de CSS-code en de bijhorende beelden zijn verschillend.
