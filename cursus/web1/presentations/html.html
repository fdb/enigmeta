HTML

HyperText Markup Language

HTML is niet ingewikkeld, maar belangrijk om te begrijpen omdat het het fundament is van de hele webpagina.

HTML

HTML bepaalt inhoud en structuur.

Het is het skelet van je webpagina, waarna je de "huid" van CSS erop legt, en het gedrag bepaalt door JavaScript.

HTML

HTML is geen programmeertaal.

Het is een opmaaktaal.

Er wordt nog wel eens gedacht dat HTML een programmeertaal is. Dat is het niet. Het is een opmaaktaal, of "markup language" in Engels. Het voegt structuur, betekenis toe aan je inhoud.

Hoe ziet HTML eruit?

<!DOCTYPE html>
<html lang="en">
<head>
	<meta charset="utf-8">
	<title>Space Monkeys Inc.</title>
	<meta name="description" content="Space Monkeys Inc is the largest provider of monkeys in space.">
	<link rel="stylesheet" href="media/style.css?v=1">
</head>
<body>
	<h1>Welcome to Space Monkeys Inc.</h1>
	<p>For years, we have been providing the best quality Space Monkeys known to man.</p>

	...

</body>
</html>

Dit is een volledig HTML document.

De kleuren hier maken niet deel uit van het document! Je past HTML aan met een gewone tekst-editor, dus niet Word, maar Notepad. Syntax coloring is enkel zodat je duidelijk onderscheid kan maken tussen de verschillende elementen van het document.

Hoe ziet HTML eruit?

Algemeen

<tagname attribute="value">
    content
</tagname>

Voorbeeld

<p id="footer" class="my-footer">
    Copyright 2010.
</p>
HTML bestaat uit "elementen". Elk element bestaat uit een open-tag met attributen. Een element kan inhoud hebben, en ook een sluit-tag. De open- en sluit-tag hebben dezelfde naam, alleen gebruik je voor de sluit-tag een slash. De inhoud-regio van een element kan een ander element bevatten. Ze zitten dan in elkaar genesteld. Zo maak je een boom, of hierarchie van elementen. Het voorbeeld toont een paragraaf-element. Het heeft een ID-attribuut en een class-attribuut. Deze worden gebruikt in CSS om elementen uit elkaar te halen, zodat je bijvoorbeeld speciale opmaak kunt instellen voor enkel de footer.

Geneste elementen

<div id="content">
    <h2>My Blog Post</h2>
    <p>Lorem ipsum...</p>
</div>
Hier is een voorbeeld van een genest stuk HTML. Het division (div)-element bevat een header (h2) en een paragraaf (p). Het h2-element en p-element zitten dus in het div-element.

Hoe HTML gebruiken?

De inhoud beschrijft de structuur.

De belangrijkste regel is dat de inhoud bepaalt hoe we de structuur van een HTML-document beschrijven.

Hoe HTML gebruiken?

Aanpak:

  • Eerst HTML: semantische structuur.
  • Dan CSS: opmaak.
De aanpak is om eerst de HTML te schrijven. Focus enkel op het kiezen van structuur-elementen die de betekenis van de inhoud bepalen. Als je daarna de CSS maakt, kan je beginnen nadenken over hoe de semantische structuur wordt opgemaakt. We gebruiken de term "semantische structuur" (semantic markup) om aan te duiden dat we focussen op de betekenis van de inhoud, niet op hoe het eruit ziet. Als je deze aanpak volgt kom je tot een stevige HTML-fundament.

HTML-elementen

Laten we eens kijken welke gereedschappen we hebben om de structuur te beschrijven.

Je hoeft die niet van buiten te kennen! Ik toon een website waarop je dit allemaal kan terugvinden.

Hoofdingen

Code

<h1>Hoofding 1</h1>
<h2>Hoofding 2</h2>
...
<h6>Hoofding 6</h6>

Resultaat

Hoofding 1

Hoofding 2

...
Hoofding 6
Eerst hoofdingen. Deze hebben niveaus, van h1 tot h6. Denk aan hoofdingen alsof je een eindwerk schrijft. Je hebt de titel van je volledige eindwerk, dit is een h1. Dan heb je onderdelen, dat zijn h2's. Dan sub-onderdelen, dat zouden h3's worden, enzovoort. Hoofdingen bepalen een hierarchie van de verschillende onderdelen van je webpagina. Je zou geen h3 gebruiken zonder eerst een h2 en h1 te gebruiken, net zoals je geen sub-onderdeel hebt van een eindwerk zonder het onderdeel en de titel van het eindwerk. Denk niet aan hoe je hoofdingen wil in een groter of kleiner lettertype. Dat is opmaak en de taak van CSS. Denk nu enkel aan de structuur van het document.

Paragrafen

Code

<p>Lorem ipsum dolor sit amet...</p>
<p>Curabitur tristique, justo et semper...</p>
<p>Proin placerat nulla a risus...</p>

Resultaat

Lorem ipsum dolor sit amet...

Curabitur tristique, justo et semper...

Proin placerat nulla a risus...

Er zijn ook paragrafen. Deze bevatten een paragraaf met tekst. Merk op dat je ook hier geen lege paragraaf laat om meer plaats te hebben tussen paragrafen: dit is iets dat je in CSS bepaalt.

Beelden

Code

	<img src="img/flower.jpg" alt="Yellow flower">

Resultaat

Om beelden in te voegen gebruik je de image tag. Ook hier kan je met CSS heel wat stylen, zoals de rand rond het beeld, de grootte, of de tekst errond moet wrappen, enz.

De "alt"-tekst, of beschrijving, is verplicht. Het staat voor "alternative", en bepaalt de tekst die in plaats van het beeld komt als het beeld niet kan worden getoond.

Zoekmachines gebruiken o.a. deze tekst wanneer je een beeld zoekt met sleutelwoorden.

Ongeordende lijsten

Code

<ul>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
</ul>

Resultaat

  • Item 1
  • Item 2
  • Item 3
Dan zijn er lijsten. Er zijn twee varianten: ongeordende lijsten, en geordende lijsten. Een ongeordende lijst bestaat uit een ul-element met daarin meerdere li-elementen. Ongeordende lijsten gebruik je wanneer de volgorde van de lijst niet belangrijk is. Een voorbeeld is een winkellijstje: je moet alle items van het lijstje meebrengen van de winkel, ze zijn dus allemaal even belangrijk.

Geordende lijsten

Code

<ol>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
</ol>

Resultaat

  1. Item 1
  2. Item 2
  3. Item 3
Geordende lijsten gebruik je wanneer de volgorde wel belangrijk is. Het enigste verschil is dat we "ol" gebruiken, voor ordered list, en niet "ul", voor unordered list. Een voorbeeld van geordende lijsten kan een recept zijn, waarin de stappen van het recept in de juiste volgorde moeten gebeuren.

Links

Code

<a href="http://www.google.com/">Google</a>

Resultaat

Google
Links maken het web tot wat het is. Wanneer de gebruiker klikt op de inhoud van de a-tag, zal de browser navigeren naar de link die staat opgegeven in het href-attribuut. In dit geval wordt de gebruiker dus doorverwezen naar de Google homepage.

Benadrukken

Code

Het eten is gewoonweg <em>verrukkelijk</em>!
Ik zou dit <strong>ten stelligste</strong> afraden.

Resultaat

Het eten is gewoonweg verrukkelijk!

Ik zou dit ten stelligste afraden.

Een benadruk-element, of "emphasis"-element, die bepaalt dat de inhoud meer moet benadrukt worden dan de omliggende inhoud. Standaard wordt de inhoud in een em-element schuin gezet. Als we een schermlezer gebruiken, een apparaat dat webpagina's luidoop voorleest (voor blinden of slechtzienden), zou de stem omhoog gaan in toon, wat dezelfde betekenis weergeeft. Een sterke nadruk-element, of "strong", bepaalt dat de inhoud nog sterker moet benadrukt worden dan een gewoon benadruk-element.

Benadrukken: zijsprong

Wat is het verschil tussen <em> en <i>?

Wat is het verschil tussen <strong> en <b>?

Wat is het verschil tussen een "italic" (i)-element en "emphasis" (em)-element? Of wat is het verschil tussen een "bold" (b)-element een "strong"-element? Zowel italic als emphasis worden op dezelfde manier weergegeven, en bold en strong worden ook op dezelfde manier weergegeven. Italic en bold verwijzen naar de opmaak van de inhoud. Emphasis en strong verwijzen naar de betekenis van de inhoud. Ze vertellen je waarom de inhoud moet getoond worden als italic of bold. Emphasis en strong zijn nieuwere elementen die zijn toegevoegd aan de HTML-taal, eens we begrepen dat het scheiden van inhoud en opmaak een goed idee was.

Tabellen

Code

<table>
    <tr>
        <th>Naam</th>
        <th>Leeftijd</th>
    </tr>
    <tr>
        <td>Danny</td>
        <td>54</td>
    </tr>
    <tr>
        <td>Lindsey</td>
        <td>37</td>
    </tr>
</table>
Een tabel bestaat uit een hele lijst van geneste elementen. Eerst is er het table-element, dat rond de hele tabel staat. Dan zijn er rijen, aangegeven door het table-row (tr)-element. In elke rij zitten tabel-cellen (td) of voor de eerste rij, tabel-headers (th).

Tabellen

Resultaat

Naam Leeftijd
Danny 54
Lindsey 37
Dit is een goede manier om een tabel te gebruiken: voor het tonen van tabulaire data. Een duidelijk teken dat dit een goede manier is, omdat er meerdere rijen zijn, met hoofdingen.

Tabellen: niet voor layout

<table>
    <tr>
        <td>Linkerkolom</td>
        <td>Middenkolom<td/>
        <td>Rechterkolom</td>
    </tr>
</table>
Hier is een slechte manier om tabellen te gebruiken. Er zijn geen hoofdingen, en enkel één rij. In plaats van tabellen te gebruiken voor tabulaire data, wordt hier een tabel gebruikt om een layout te maken met 3 kolommen. De maker van deze pagina misbruikt het feit dat cellen als vakken renderen van links naar rechts. Maar de layout wordt bepaald door de opmaak, en hoort dus niet thuis in HTML. De layout is de taak van CSS.

Geneste elementen

Slecht voorbeeld: doe dit niet!

<p>Bekijk de <a href="...">resultaten</p></a>

Kan je de fout vinden? De sluit-tags staan in de verkeerde volgorde.

Je browser zal niet crashen, maar zal zijn best doen om dit juist te tonen.

In ingewikkeldere structuren moet de browser soms "gokken" wat je bedoelt, en dat kan leiden tot onverwachte resultaten. Check altijd of je tags goed zijn afgesloten, en gebruik een validator om dit na te kijken.

Regels rond nesten

Slecht voorbeeld: doe dit niet!

<ul>
    <p>Paragraaf</p>
    <h1>Hoofding</h1>
</ul>
Je kan niet eender welk element binnen eender welk ander element zetten. Binnen een ul en ol kan je bijvoorbeeld enkel li-elementen gebruiken. De browser ontploft niet als je dit toch doet. Hij zal z'n best doen om alles weer te geven, maar het kan leiden tot onverwachte resultaten en problemen die moeilijk op te sporen zijn.

Zie je hier elementen tussenstaan die enkel gaan over opmaak en niet over inhoud? Bijvoorbeeld de font-tag: dit element gaat enkel over de presentatie , het kiezen van een lettertype. Eigen elementen?

Kan ik zelf HTML elementen verzinnen?

<pony>
	<eyes>adorable</eyes>
	<tail>cute</tail>
</pony>

Je kan geen eigen HTML verzinnen. De browser zou immers niet weten wat hij ermee moet aanvangen. Als je een "smell"-element verzint, hoe moet de browser dan weten dat hij er geur aan moet koppelen? Wat je wel kan doen is elementen een class of id meegeven.

Class Attribute

<div class="nieuwsbericht">
	<h2>Nieuwe tienkoppige Elvis ontdekt</h2>
	<p>In de Amerikaanse staat Tennessee...</p>
</div>

Resultaat

Nieuwe tienkoppige Elvis ontdekt

In de Amerikaanse staat Tennessee...

Zo wordt dit normaal gedaan: je duidt aan dat een bepaald blok een "nieuwsbericht" is. In CSS kan je dan helemaal bepalen hoe zo'n nieuwsbericht eruit ziet.

ID vs Class

<div id="header">
	<h1>Eenwielers Peeters</h1>
</div>
<div id="content">
	<div class="fiets">
		<h2>Model Alpha</h2>
	</div>
	<div class="fiets">
		<h2>Model Omega</h2>
	</div>
</div>
Wat is het verschil tussen een ID (identifier) en class? Een ID bepaalt de identiteit van één enkel element op de pagina. Een class, of klasse bepaalt een type van opmaak. Van een ID kan er maar één zijn, bijvoorbeeld één pagina-hoofding, navigatie, footer. Van een class kunnen er meerdere zijn, bv. verschillende fietsen, nieuwsberichten, ...

HTML versies

Er zijn verschillende versies van HTML.

De doctype bepaalt de HTML-versie.

Bovenaan het document kan je de browser zeggen welke versie van HTML je schrijft, en de browser kan die informatie gebruiken om een andere "renderer" te kiezen. Dit klinkt allemaal ingewikkeld, maar er is één regel:

HTML 5

<!DOCTYPE html>
Je hoeft enkel deze versie te onthouden. Dit is de minimale HTML 5 doctype. Het is belangrijk omdat de aanwezigheid van de doctype bovenaan je HTML-code tegen browsers zegt dat ze je pagina moeten renderen in "standaard"-modus. Deze modus is de meest recente, en volgt de CSS-specificatie zo juist mogelijk. Als je geen doctype hebt zullen sommige browser je pagina renderen in "quirks"-mode (eigenaardigheden-modus). Deze modus volgt de CSS-specificatie niet en kan dus voor heel wat frustaties zorgen. Als je problemen hebt met het renderen van de pagina, is het eerste wat je moet nakijken of je een doctype hebt.
