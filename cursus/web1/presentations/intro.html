Bottom-up aanpak

  • Zelfredzaamheid
  • Samenwerking

We hebben er speciaal voor gekozen om je HTML en CSS te leren zodat je echt weet hoe pagina's gemaakt worden, en wat de code erachter is. Dit is belangrijk om verschillende redenen.

Je zou, in plaats van met HTML te werken, ook met een grafische editor werken. Die schrijft dan de HTML voor jou. Fantastisch!

Tot er iets mis gaat. De editor vormt een dun laagje boven de HTML en CSS: hij abstraheert als het ware de HTML en CSS tot een grafische omgeving.

Echter, door te doen alsof HTML en CSS niet bestaan loop je tegen problemen aan die je zelf niet kan oplossen. Een element staat niet op de juiste plaats? Als je nooit de HTML en CSS ziet weet je ook niet waar je moet beginnen zoeken.

Daarom is het belangrijk om te begrijpen hoe webpagina's gemaakt worden, alvorens je visuele tools gebruikt om pagina's aan te maken.

In de "echte" wereld zal je grote websites waarschijnlijk in team maken. Als je samenwerkt met een programmeur die de HTML en CSS voor jou doet is het nog steeds belangrijk om te begrijpen wat er mogelijk is. Dit laat de conversatie tussen jou en andere teamleden vlotter verlopen.

Alternatieven

Waarom niet Flash?

  • Toegankelijkheid
  • Zoekmachines

Veel designer-sites zijn in Flash gemaakt. Flash is ideaal voor animatie, geluid, video, dus waarom doen we niet alles in Flash?

Er zijn een aantal redenen waarom dit geen cursus Flash is.

De hoofdreden is toegankelijkheid. Mensen die niet in staat zijn van visuele webpagina's te bekijken (zoals blinden) kunnen je flash-website niet bezoeken. Dit is misschien niet belangrijk als je een portfolio met grafisch werk maakt, maar wordt wel belangrijk als je websites ontwikkeld voor mensen die.

Daarnaast zijn er de zoekmachines. Een site in Flash bevat bijna geen zinnige informatie voor zoekrobots, waardoor je website het erg povertjes zal doen in Google.

Alternatieven

Waarom niet Flash?

Tenslotte zijn er bezoekers die met een mobiele webbrowser de pagina bekijken, zoals de iPhone of iPad.

Deze apparaten ondersteunen geen Flash, en het is duidelijk uit Apple's standpunt dat dit in hun ogen een verouderde technologie is die ze niet willen ondersteunen.

Voordelen

  • Toegankelijkheid
  • Overdraagbaarheid
  • Onderhoudbaarheid
  • Snelheid
Door HTML/CSS/JavaScript te scheiden. 1. Propere, semantische HTML heeft voordelen voor gebruikers die niet-visuele webbrowsers gebruiken, zoals blinden. Waarom die focus op blinden? Het web is een democratische tool. Maakt niet uit welke huidskleur je hebt, welke webbrowser je gebruikt of welke handicap je hebt. Daarnaast zijn er ook zoekrobotten die je website opzoeken, en die kunnen ook niet zien. Google leest je site en gebruikt vooral de structuur-informatie uit HTML om te bepalen wat belangrijk is. 2. Je pagina wordt makkelijker overdraagbaar. Door HTML/CSS te scheiden kan je een nieuwe stylesheet implementeren en je hebt een mobiele versie van je pagina. We kunnen ook delen tussen verschillende projecten herbruiken. 3. Je pagina's worden makkelijker te onderhouden. Een designer kan aanpassingen aan de CSS doen zonder de HTML te hoeven aan te passen of de JavaScript code te veranderen. 4. Door HTML en CSS te scheiden moeten bezoekers van je website de CSS maar één keer downloaden, en gebruiken daarna de versie die ze in cache bewaren, wat de site sneller maakt.
