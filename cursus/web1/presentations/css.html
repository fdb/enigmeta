CSS?

Cascading Style Sheets

CSS dient om de opmaak te bepalen van elementen die je in HTML hebt bepaalt. Alle webpagina's op het web gebruiken CSS om opmaak te doen, dus je hebt heel veel mogelijkheden. Net zoals HTML is CSS ook geen programmeertaal, maar een beschrijvingstaal.

Mogelijkheden

  • Kleur
  • Lettertype
  • Grootte
  • Marges
  • Randen
  • Achtergronden
  • Layout
CSS kan gebruikt worden om de kleur van elementen te bepalen, het lettertype, de grootte van elementen, marges en randen, achtergronden en layout. En nog veel meer!

Voorbeeld

h1 {
	font-size: 18pt;
	color: #f34466;
}

#footer a {
	color: green;
	border: 1px solid #553244;
}
Dit is een voorbeeld van CSS. Op deze pagina staan twee CSS-regels. Elke CSS-regel bestaat uit drie delen.

Voorbeeld

Selector (Keuzeschakelaar?)

1. Eerst selecteren we het element.

p {
	color: red;
}

Voorbeeld

Properties (Eigenschappen)

2. Dan zeggen we wat we willen veranderen.

p {
	color: red;
}

Voorbeeld

Values (Waarden)

3. Dan waar we het naar willen veranderen.

p {
	color: red;
}
De combinatie van een eigenschap en waarde heet een declaratie (declaration). Je kan meerdere declaraties hebben per CSS-regel. Hier maken we alle paragrafen op de pagina rood.

Opbouw: Anatomie van CSS

p {
	color: red;
}
Een aantal belangrijke andere elementen: Eerst komt de selector. De eigenschappen en waarden van die selector staan tussen accolades. Na elke waarde komt de eigenschap, gevolgd door een dubbele punt, en dan de waarde. Elk paar van eigenschap en waarde sluit je af met een puntkomma.

Opbouw: Meerdere eigenschappen

p {
	color: red;
	font-size: 12px;
}
Meerdere eigenschappen zet je gewoon onder elkaar. Na elk ervan komt dus de puntkomma.

Opbouw: witruimte

Witruimte is niet belangrijk.

p { color: red; background: black; }
Witruimte doet er niet toe, zolang je maar de juiste accolades, dubbele punten en puntkomma's toevoegt.

Volledig voorbeeld

body { 
	background: red; 
	font: 10px Verdana, sans-serif; 
}

#container {
	background: white; 
	width: 900px; 
	margin: 0 auto;
}

h1 { 
	font: 15px Georgia, serif;
}
Hier is een volledig voorbeeld. Dit zou een volledig CSS document kunnen zijn. Drie CSS declaraties, met elke een aantal eigenschappen met waarden. Elke CSS document zal er zo uit zien.

Hoe linken?

3 manieren om CSS aan HTML te linken

  • In het element (Inline)
  • Ingebed (Embedded)
  • Extern (External)
Voor we dieper ingaan op CSS, gaan we eerst uitleggen hoe we CSS kunnen koppelen aan een HTML document. Er zijn 3 manieren om CSS aan HTML te linken, gaande van "makkelijker" naar "beter".

Inline CSS (BAD)

	<p style="color: red;">Hello</p>

Resultaat

Hello

Dit is inline CSS. De CSS staat in het "style" attribuut van een HTML-element op de pagina. Dit wil zeggen dat de CSS enkel terugslaat op deze. Denk eraan, we willen een scheiding tussen CSS en HTML. Deze vorm mengt CSS en HTML. Het betekent ook dat we stijlen niet kunnen herbruiken. Als we achteraf de stijl willen veranderen moeten we door alle pagina's om de kleur van de paragrafen aan te passen.

Embedded CSS (OK)

<head>
	<style type="text/css">
		p { 
			color: red; 
		}
	</style>
</head>
<body>
	<p>Hello</p>
</body>

Resultaat

Hello

Hier gebruiken we het style element in de header. Daartussen kunnen we CSS plaatsen. Dit is OK voor ontwikkeling en als je de CSS maar op één pagina gebruikt.

Externe CSS (Goed!)

about.html

<head>
	<link rel="stylesheet" href="style.css">
</head>

style.css

p {
	color: red;
}
Elk serieus gebruikt betekent dat we de CSS naar een extern bestand gaan verplaatsen. We gebruiken een link-element om naar het CSS-document te verwijzen. Je browser kan dit bestand downloaden en "cachen", dus opslaan in het geheugen. Op andere pagina's hoeft je browser die zelfde stylesheet dan niet meer te downloaden. Dit maakt je pagina's sneller, omdat je niet elke keer deze CSS-file van het Internet moet downloaden. Daarnaast is het ook eenvoudig om de stijl aan te passen: je hoeft enkel die file aan te passen, en alle pagina's krijgen automatisch de nieuwe stijl.

Deel 1: CSS Anatomie

  1. Selector
  2. Declaratie
  3. Cascade
We gaan nu dieper in op items in de CSS anatomie, of "syntax", om ervoor te zorgen dat je het echt begrijpt.

Selector

(Het ding voor de accolades)

p {
	color: red;
}
Eerst praten we over de selector. Dit staat voor de accolades en bepaalt op welke elementen in het HTML-document de regel toepassing heeft. Een selector selecteert. Het bepaalt welke elementen in het HTML-document we willen selecteren om er stijl-aanpassingen op toe te passen. Als we bijvoorbeeld alle hoofdingen in 18 punt willen zetten, moeten we eerst een manier hebben om die te selecteren. Daarvoor gebruiken we de selector. In dit voorbeeld slaat de CSS-regel op alle paragrafen, en worden dus alle paragrafen in het HTML-document rood.

Manieren om te selecteren

  1. Element
  2. Class
  3. ID
  4. Combinaties
Er zijn vier manieren om selecties te maken.
  1. Op basis van een element, de tag-naam dus. Het vorige voorbeeld met de "p" selector werkt op basis van het element.
  2. Op basis van een class, de klasse van een element. Dit kan je zelf bepalen. Je kan meerdere elementen dezelfde klasse geven.
  3. Op basis van een ID, identifier. Dit kan je ook zelf bepalen. De ID is uniek.
  4. Op basis van de positie in het document, een combinatie van de bovenstaande.

1. Element

p { }

Selecteer alle <p> elementen in het hele document.

Dit zegt: selecteer alle paragraaf-elementen in het hele document. Eender waar op de pagina.

2. Class

.newsitem { }

	<div class="newsitem">
		...
	</div>

Selecteer elk element met de klasse-naam "newsitem".

Het punt "." betekent dat je op klasse-naam selecteert.

3. ID

#footer { }

	<div id="footer">
		...
	</div>

Selecteer elk element met de identifier "footer".

ID's zijn uniek, dus dit zou maar één element mogen selecteren.

Het hekje "#" betekent dat je op id-naam selecteert.

4. Combinaties (1)

Selector

#about p { }

HTML

<div id="about">
	<h2>About this page</h2>
	<p>Lorem ipsum...</p>
	<p>Dolor es sit...</p>
</div>
<p>Nullam non risus lacinia</p>

Selecteer elk p element dat een afstammeling is van elk element met een id-attribuut gelijk aan "about".

Een afstammeling betekent dat het element binnen een ander element komt. Denk eraan: de elementen staan in hiërarchie, een boom die je definiëert in HTML. In dit geval staat het div-element met de "about" id bovenaan, en daarin zitten de twee p-elementen. De laatste p-element wordt niet geselecteerd omdat hij niet een afstammeling is van een element met "about" als id.

4. Combinaties (2)

Selector

#about p { }

HTML

<div id="about">
	<div class="something">
		<p>Lorem ipsum...</p>
	</div>
</div>

div#about
  |
  +-- div.something
        |
        +-- p
        +-- p
In CSS is de betekenis van HTML dat het element er binnen moet zitten, maar niet direct erbinnen, je kan ook elementen er tussen hebben.

4. Combinaties (3)

Selector

p.intro { }

HTML

<div id="about">
	<p class="help intro">Lorem ipsum...</p>
	<p class="help">Dolor es sit...</p>
</div>

Selecteer elk p element met een klasse die het woord "intro" bevat.

Let op: geen spatie tussen "p" en "intro". Dit gaat over één element met die twee eigenschappen: een p-tag en een klasse "intro". Dezelfde tag.

Subtiel verschil

Een klasse-attribuut bevat.

vs.

Een id-attribuut gelijk aan.

Hiervoor spraken we over een klasse-attribuut dat het woord bevat. Tegenover een id-attribuut die gelijk is aan een bepaalde waarde.

Subtiel verschil

.intro { background: green; }
.help { color: red; }

<p class="help intro">hi</p>
<p class="help">hi</p>
Het HTML klasse-attribuut kan verschillende waarden hebben. Elke waarde is één woord. De HTML-lijn "help intro" is interessant, omdat de twee CSS-regels deze lijn selecteren. De intro-regel bepaalt dat de achtegrond groen moet zijn. De help-regel bepaalt dat de kleur rood moet zijn. In de eerste "p" kunnen de twee regels "matchen", toegepast worden. Het eindresultaat is dat de inhoud een groene achtergrond krijgt, en een rode tekst-kleur. In de tweede "p" matcht enkel de help-selector, dus enkel de tekst wordt rood Ik toon later wat er gebeurd als je dezelfde eigenschap twee keer definieert, dus bijvoorbeeld eerst de kleur blauw maakt en dan de kleur rood.

Nog eentje!

Pseudo klasses

a:link { } /* onbezochte link */
a:visited { } /* bezochte link */
a:hover /* muis over link */
a:active /* geselecteerde link */
Pseudo klasses zijn een ander soort selectors, waar je de dubbelpunt gebruikt om te selecteren op "status" (net zoals je de punt gebruikt om te selecteren op klasse). Je gebruikt ze vooral op link-elementen (de "a", anchor-tag) om aan te duiden wat er gebeurt als je over een link gaat met de muis, de "hover"-status (state).

Declaration

(Het ding tussen de accolades)

p {
	color: red;
}
Nu praten we over de declaratie. De verschillende declaraties staan tussen de accolades. De declaratie kan een onbeperkt aantal paren bevatten van een eigenschap en een waarde. Ze komen steeds met twee voor: eerst de eigenschap, dan de dubbele punt, dan de waarde. Na elke declaratie komt een puntkomma om aan te geven dat dit het einde is van de declaratie, en dat de volgende declaratie mag komen.

Declaration

Een declaratie is een combinatie van een eigenschap en een waarde.

Je declareert wat je wil veranderen.

Een declaratie is een combinatie van een eigenschap (property) en waarde (value). Je declareert, vertelt, wat je wil veranderen en in wat je het wil veranderen.

Eigenschappen

CSS bepaalt de dingen die je kan veranderen:

  • font
  • color
  • background-color
  • border
CSS heeft een vaste lijst van dingen die je kan veranderen. Voorbeelden zijn het lettertype, de kleur van de tekst, achtergrond-kleur, rand enz. Dit zijn de eigenschappen, properties.

Waarden

Elke eigenschap heeft een lijst van waarden:

	font: 12px Verdana, sans-serif;
	color: red;
	background-color: #112233;
	border: 1px solid rgb(193, 193, 193);
Elke eigenschap heeft mogelijke waarden die van toepassing zijn op die eigenschap. Voor een lettertype (font) kan je bijvoorbeeld de grootte en de naam van het lettertype kiezen. Voor de kleur van de tekst en de achtergrondkleur kan je een kleur-waarde ingeven. Er zijn verschillende manieren om kleuren aan te geven. Color gebruikt een "systeemkleur" met een vaste naam, zoals rood, of groen. Background-color gebruikt een hexadecimale waarde die bestaat uit een rood, groen en blauw component. En bij border zie je de kleur in RGB-waarden. Als een eigenschap een kleur verwacht, kan je elk van die manieren gebruiken om kleuren te gebruiken. Color hoeft dus niet altijd met systeemkleuren, en background-color kan niet enkel hexadecimale waarden gebruiken. Border heeft ook nog de dikte van de boord (1 pixel) en het soort rand (solid).

De cascade

De cascade bepaalt welke declaraties van toepassing zijn op een element.

De "cascade" is de "C" in "CSS".

Cascade volgorde

Dingen worden toegepast in de volgorde waarin je ze definiëert.

	p { color: red; }
	p { color: blue; }

De laatste declaratie wint.

De eerste regel die je moet onthouden is dat declaraties die later komen in het bestand "winnen" over declaraties die eerder komen. In dit geval komt blue na red, dus alle paragrafen krijgen blauwe tekst.

Cascade: hoe specifieker hoe beter

Maar...

Als een element specifieker is, wint het ook.

Dus het laatste en meest specifieke element wint.

Zeggen dat het laatste element altijd wint is niet helemaal juist. Als een declaratie specifieker is, wint het. Dus het laatste en meest specifieke element wint.

Cascade: hoe specifieker hoe beter

	<div id="alpha">
		<div id="beta">
			<h1>Hello</h1>
		</div>
	</div>

	#alpha #beta h1 { color: red; }
	#alpha h1 { color: blue; }
	h1 { color: green; }

Welke selector is specifieker?

Welke kleur heeft de header?

De eerste selector wint, omdat het de meest specifieke is, zelfs als de tweede regel erna komt. Omdat we verwijzen naar een div met id alpha, een div met id beta, en een header, zijn we specifieker dan als we verwijzen naar enkel een div met id alpha, of zelfs enkel een h1-element.

Cascade: regels rond specificiteit

Er zijn regels die bepalen welke declaratie wint.

Deze regels werken met punten.

Dit is niet zomaar gokwerk om te bepalen welke declaratie gaat winnen. Gelukkig maar! Er zijn regels die bepalen welke declaratie wint. De manier hoe dit werkt is met punten. Dit gaat vrij diep, dus even opletten!

Cascade: regels rond specificiteit

  • Element: 1 punt
  • Class: 10 punten
  • Id: 100 punten
Als er een element (p, h1) in de selector staat krijgt de regel 1 punt. Staat er een class in, krijgt de regel 10 punten. Staat er een id in, krijgt de regel 100 punten. Deze punten worden opgeteld.

Cascade: regels rond specificiteit

	<div id="container">
		<div class="newsitem">
			<h2>Goed Nieuws!</h2>
		</div>
	</div>

	#container h2 { color: red; } /* 100 + 1 = 101 */
	.newsitem h2 { color: blue; } /* 10 + 1 = 11 */
Hier is een voorbeeld. De eerste regel bevat een id-selector en een element, goed voor 100 + 1 = 101 punten. De tweede regel bevat een class-selector en een element, goed voor 10 + 1 = 11 punten.

Cascade: regels rond specificiteit

Geen paniek.

Dit lijkt allemaal heel ingewikkeld, maar meestal hoeft dit geen probleem te zijn. In de voorbeelden die we tegenkomen ga je nooit hoeven te rekenen hoeveel punten je hebt. Het systeem is zo gemaakt dat het "intuïtief" werkt, dus in bijna alle omstandigheden doet wat je verwacht. Het is wel goed om in het achterhoofd te houden dat id-selectors "krachtiger" zijn dan class-selectors, die op hun beurt weer krachtiger zijn dan gewone element-selectors.

Overerving (inheritance)

Elementen kunnen stijlen van bovenliggende elementen overerven.

Overerving

	#blog { color: red; }

	<div id="blog">
		<p>Hello</p>
	</div>
Hier is een voorbeeld. De blog-div krijgt de kleur "rood". Het onderliggende element "p" erft de waarde van het bovenliggende element blog, en krijgt dus de kleur rood. Concreet betekent dit dat alle tekst in de blog-div de kleur krijgt van de blog-div tenzij je anders specifiëert.

Niet alles wordt overgeërfd.

Tekst-gerelateerde eigenschappen worden overgeërfd.

Layout-gerelateerde eigenschappen worden niet overgeërfd.

Dit klinkt simpel, maar niet alles wordt overgeërfd. En maar goed ook. De regel is dat alle eigenschappen gerelateerd aan tekst, zoals kleur, font, uitlijning enz. wordt overgeërfd. Layout-gerelateerde eigenschappen, zoals randen, marges, positie worden niet overgeërfd. Dit heeft voordelen, maar net zoals bij de cascade is het systeem zo gemaakt dat het "intuïtief" werkt. Je hoeft hier niet bewust rekening mee te houden omdat het systeem doet wat je verwacht.

Nu gaan we wat dieper in op een aantal eigenschappen van CSS die we voor de gemakkelijkheid hebben over geslagen.

Eigenschappen: granulariteit

Eigenschappen kan je op verschillende manieren schrijven

	border: 1px solid rgb(193, 193, 193);

	border-width: 1px;
	border-style: solid;
	border-color: rgb(193, 193, 193)
Eigenschappen kan je op verschillende manieren schrijven. Dit is de granulariteit van een property. Border bv. bevat zowel de eigenschap border-width, border-style en border-color. Het is een kortere versie (shorthand) van de drie eigenschappen achter elkaar. Belangrijk is dat je bij de kortere versie de eigenschappen in de juiste volgorde zet. Je kan niet zomaar eerst de style zetten en dan de width.

Eigenschappen: granulariteit

	border-left: 1px solid rgb(193, 193, 193);
	border-right: 2px dashed rgb(255, 0, 0);
	
	border-bottom-color: red;
Je kan de border nog meer uit elkaar trekken, en het hebben over enkel de linker-rand, of de rechter-rand. Als je gewoon "border" zegt heb je het over de hele rand.

Eigenschappen: granulariteit

	font: italic small-caps bold 10pt Verdana, sans-serif;
	
	font-style: italic;
	font-variant: small-caps;
	font-weight: bold;
	font-size: 10pt;
	font-family: Verdana, sans-serif;
Hier zie je een voorbeeld met "font". Alle verschillende eigenschappen onderaan zitten in één declaratie bovenaan. De volgorde is belangrijk: eerst style, dan variant, dan weight, dan size, dan family. Ik kan die ook niet onthouden, en meestal heb je ze ook niet allemaal nodig. Je kan dit opzoeken (bv. op w3schools) maar ik onthoud gewoon eerst de grootte en dan de familie. Dat is meestal genoeg. De rest kan je dan nog instellen met "lange" eigenschappen, zoals font-variant.

Eigenschappen: granulariteit

	p { border: 1px solid red; }
	p.intro { border-color: blue; }
Het handige aan granulariteit is dat je specifieke eigenschappen kan veranderen. Bijvoorbeeld hier: wat is de grootte van de rand van p-elementen? En wat is de randkleur van p.intro? Als je dan besluit van alle randen 2 pixels dik te maken, volgt ook de rand van p.intro, omdat die de randdikte "overerft" (inherit) van p.

Maateenheden

Maateenheden (units) zijn de manieren waarop groottes worden uitgedrukt.

  • em (ems)
  • ex (x-height)
  • px (pixels)
  • % (percentages)
  • in (inches)
  • cm (centimeters)
  • mm (millimeters)
  • pt (points)
  • pc (picas)
Maateenheden zijn manieren waarop je grootte kan aanduiden. CSS kent er verschillende: je herkent waarschijnlijk centimeters, pixels en percentages.

Maateenheden: absoluut vs relatief

px vs %
Je hebt twee types van maateenheden in CSS: absolute maateenheden en relatieve maateenheden.

Absolute Maateenheden

pixels (px) zijn absolute maateenheden.

Ze hebben altijd dezelfde grootte.

	p { font-size: 12px; }
Een voorbeeld van absolute maateenheden zijn pixels. Als je een maat, bijvoorbeeld font-size, in pixels specifieert, krijg je altijd hetzelfde resultaat.

Relatieve Maateenheden

Percentages (%) zijn relatieve maateenheden.

Hun grootte hangt af van andere elementen.

	body { font-size: 12px; } /* 12 pixels */
	h1 { font-size: 200%; } /* 24 pixels: 12 * 2 */
	h1 a { font-size: 75%; } /* 18 pixels: 12 * 2 * 0.75 */
Een voorbeeld van relatieve maateenheden zijn percentages. Ze bepalen de grootte in relatie tot andere elementen. In dit voorbeeld geven we de body een font-grootte van 12px, een absolute maat. Dan definiëren we h1 met een font-grootte van 200%. De grootte van h1 is dus 24 pixels: de font-size van de body * 2. Links in een h1 zijn 18 pixels: de grootte van h1 (24 pixels) * 0.75. De grootte wordt dus berekend aan de hand van andere CSS-declaraties die er toepassing op hebben. Dit heeft voordelen. Als je de body declareert als absolute maat en al de rest als relatieve maten, kan je makkelijk alle elementen op de pagina groter of kleiner maken door één declaratie aan te passen, namelijk de body.

De font-family eigenschap

  • Andale Mono
  • Arial
  • Comic Sans MS
  • Courier New
  • Georgia
  • Impact
  • Times New Roman
  • Trebuchet MS
  • Verdana
De mogelijke lettertypes hangen in eerste instantie af van welke lettertypes gebruikers op hun eigen computer hebben geïnstalleerd. Dit betekent dat je enkel de grootste gemene deler van alle fonts kan gebruiken. Deze lijst is erg beperkt, daarom dat je op de meeste websites steeds dezelfde lettertypes ziet terugkomen. Dit is de lijst van "core fonts". Hier kan je van uit gaan dat iedereen ze heeft geïnstalleerd.

De font-family fallback

	p { font-family: "Lucida Grande", Arial, sans-serif; }
CSS biedt de mogelijkheid om alternatieven te declareren als een lettertype niet kan worden gevonden ("fallback"). Hier zie je dat alle paragrafen in Lucida Grande moeten komen. Dit lettertype is standaard enkel op Mac beschikbaar, dus we definiëren het alternatief arial (dat normaal overal is geïnstalleerd). Bestaat dit ook niet, dan specifiëren we "sans-serif", een niet nader genoemd lettertype zonder schreven.

De font-family fallback

  • sans-serif
  • serif
  • monospace
De ultieme fallback-lettertypes zijn sans-serif, serif, en monospace. zien er niet overal hetzelfde uit, maar de stijl is wel consequent. Sans-serif is zonder schreven, Serif is met schreven, Monospace is een blokletter waarbij elk teken dezelfde breedte heeft. Deze gebruik je voor code-voorbeelden, zoals in deze slides.

Web fonts

Web fonts doorbreken de sleur!

Sinds kort kan je echter ook web fonts specifiëren die, net zoals beelden, worden gedownload van de server en gebruikt. Dit is leuk omdat je dan eender welk lettertype kan gebruiken op je website zonder dat je je zorgen hoeft te maken of de gebruiker dit lettertype heeft geïnstalleerd. Vroeger moest je beeldjes genereren van hoofdingen om ze in een ander lettertype te zien. Web fonts zijn echte lettertypes: je kan de tekst selecteren, een ander lettertype gebruiken voor print, enz.

Google font directory

Er zijn wel beperkingen naar gebruikersrechten. Net zoals je niet zomaar eender welk beeld in een werk mag gebruiken, mag je ook niet zomaar eender welk lettertype op je website gebruiken, enkel één waartoe jij de rechten hebt. De fonts op Google font directory hebben allemaal heel vrije gebruiksrechten, en je mag ze dus op eender welke website gebruiken met eender welk doel. Een webfont gebruiken is erg eenvoudig

Deel 2: Layout

We hebben het nu vooral over het systeem van CSS gehad. Laten we het over de layout hebben. Dit is hoe de declaraties in CSS toepassing hebben op hoe ze in de browser worden getoond.

Box Model

Het box model is een belangrijk concept in CSS.

Het bepaalt de relatie tussen:

  • Inhoud (Content)
  • Vulling (Padding)
  • Rand (Border)
  • Marge (Margin)
Als je één ding moet weten over layout in CSS, is het het box model.

Box Model

Hier is het box model. Het middelste deel is de inhoud. Dit is de tekst van een paragraaf, net tot aan de randen van de tekst. Dit is de kleinste rechthoek waarbinnen je de tekst kan bevatten. Daarrond staat de vulling, of padding. De padding geeft de content wat meer ruimte tussen zichzelf en de rand. De vulling is normaal transparent, maar dat kan je veranderen De rand is hetgeen je tekent. Daarvan kan je de dikte, kleur en stijl bepalen. Rond de rand staat de marge. Dit is de ruimte tussen dit element en het volgende element. De marge is altijd transparent.

Box Model Voorbeeld (1)

content
div { 
	background: #5BA1CF;
}
Hier is een voorbeeld van een element met enkel een achtegrond-kleur, zonder padding, border of margin. Merk op dat de achtegrond net tot aan de randen van de tekst komt.

Box Model Voorbeeld (2)

content
div { 
	background: #5BA1CF;
	padding: 10px;
}
Hier hebben we vulling (padding) aangebracht. De achtergrond loopt dus mee met de padding, tot aan de rand.

Box Model Voorbeeld (3)

content
div { 
	background: #5BA1CF;
	padding: 10px;
	border: 2px solid red;
}
Rond de vulling komt de rand.

Box Model Voorbeeld (4)

content
div { 
	background: #5BA1CF;
	padding: 10px;
	border: 2px solid red;
	margin: 20px;
}
De marge bepaalt de afstand tot andere elementen. In dit geval, het afstand tot de rand van de slide. Merk op dat de achtegrond niet buiten de rand komt tot in de marge. De marge is altijd transparent.

Individuele marges

content
div {
	padding-top: 5px;
	padding-right: 10px;
	padding-bottom: 50px;
	padding-left: 0;
}

/* of */

div {
	padding: 5px 10px 50px 0;
}
Je hoeft niet alle vulling, randen of marges dezelfde grootte te geven. Je kan specifieke waarden gebruiken voor de rechter-, linker-, boven- en ondermarge.

Display

Display is één van de belangrijkste eigenschappen van CSS.

  • display: inline;
  • display: block;
Display bepaalt de manier waarop elementen worden getoond. Er zijn twee waarden voor de display: inline en block. Er zijn er nog een hele hoop meer, maar dit zijn de enige twee die je waarschijnlijk zal tegenkomen.

display: inline;

Inline vakken gaan mee met de "flow" van je pagina.

elk element is een inline element . Inline elements gaan mee met de "flow" van je pagina .

Voorbeelden

  • <a>
  • <span>
  • <img>
  • <input>
Inline "vakken" (boxes) kunnen naast elkaar staan en vloeien standaard mee van links naar rechts. Voorbeelden van elementen die standaard op inline staan zijn anchors, span-elementen, image-elementen of input-elementen. De default stylesheet declareert dus aan die elementen als display: inline;. Span is het generieke inline element. Elk van de elementen hierboven is omgeven door een span-tag. Zoals je uit het voorbeeld wel merkt kan je het dus ook zien als tekst. Een paragraaf bevat een aantal woorden, en elk woord op zich wordt door je browser als een inline vak gerenderd. Daarom breekt tekst dus ook af na het einde van de lijn.

display: block;

Block vakken nemen de hele regel in.

Voorbeelden

  • <h1>
  • <p>
  • <div>
Block "vakken" nemen alle ruimte van de lijn in. Elk element dat daarvoor of daarna komt moet dus beginnen op een nieuwe regel. Voorbeelden zijn headers, paragrafen en div-elementen. Het div is het generieke block element. Elk van de elementen hierboven is omgeven door een div-tag.

Display voorbeeld

  <p>
    Gelieve te <a href="...">zoeken</a>
  </p>

Hier is een voorbeeld. De anchor-tag is standaard een element met display: inline, en wordt dus binnen de paragraaf getoond.

Gelieve te zoeken

Maar wat gebeurd er als we de anchor stylen met display: block? De anchor tag heeft nu z'n plaats nodig, en verplaatst dus naar de volgende regel. Weeral, block-elements kunnen niet samen met andere elementen op dezelfde regel staan. Je zou dit ook met een break-element (br) kunnen doen, maar hier hoeven we niets aan de HTML te veranderen om hetzelfde effect te verkrijgen. We kunnen dus achteraf in CSS nog kiezen hoe we de elementen op de pagina tonen.

Position

  • static
  • relative
  • absolute
  • fixed
Position bepaalt de positie van het element ten opzichte van andere elementen op de pagina. Het laat ons toe om alles wat we net hebben geleerd overhoop te gooien! Dit is moeilijk om in slides te tonen, dus we zullen dit zometeen zien in de browser.
